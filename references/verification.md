# 交叉验证框架

## 一、来源可信度评估

| 维度 | 高可信度 | 低可信度 |
|------|---------|---------|
| **可追溯性** | 署名作者、明确来源 | 匿名、"据知情人透露" |
| **利益冲突** | 独立第三方 | 利益相关方（如竞争对手） |
| **历史记录** | 一贯准确 | 多次造谣/反转 |
| **查证意愿** | 提供证据、欢迎核查 | 拒绝质疑、删帖控评 |
| **多元性** | 多个独立来源确认 | 单一信源 |

## 二、信息一致性检查

**三向验证法：**

| 来源类型 | 检查内容 |
|---------|---------|
| **官方信源** | 公告、财报、备案文件 |
| **第三方** | 媒体报道、监管机构 |
| **市场信号** | 股价、交易量、预测市场 |

**一致性矩阵示例：**

```
              官方公告  权威媒体  员工爆料  股价反应  预测市场
官方公告         ✓       ✓       ?       ✓       ✓
权威媒体         ✓       ✓       ✓       ✓       ✓
员工爆料         ?       ✓       ✓       ?       ✓
股价反应         ✓       ✓       ?       ✓       ✓
预测市场         ✓       ✓       ✓       ✓       ✓
```

## 三、矛盾点识别

当信息出现矛盾时：

1. **记录矛盾点** - 明确哪些来源说法冲突
2. **追溯原始信源** - 找到"一手信息"而非二手转述
3. **核查证据链** - 每个说法有什么证据支撑？
4. **分析动机** - 不同来源是否有利益驱动？
5. **寻找佐证** - 是否有其他间接证据支持某一方？
