# 调查方法论

## 一、问题定义

在开始调查前，先明确：

| 要素 | 说明 | 示例 |
|------|------|------|
| **核心问题** | 你到底想知道什么？ | "X 公司是否真的裁员 30%？" |
| **时间范围** | 关注什么时间段的信息？ | "最近 1 个月" |
| **地理范围** | 关注哪些地区？ | "中国区" |
| **信息类型** | 需要什么类型的证据？ | "官方公告、员工爆料、财经数据" |

## 二、信息源选择策略

| 信息类型 | 优先选择 | 避免 |
|---------|---------|------|
| **事实数据** | 官方公告、财报、权威媒体 | 匿名论坛、小道消息 |
| **市场反应** | 股价、交易量、预测市场 | 单一自媒体解读 |
| **民间声音** | 多个独立信源交叉验证 | 单个"内部人士" |
| **历史记录** | Wayback Machine、存档站 | "我记得"式口述 |

## 三、调查流程

```
1. 问题定义 → 2. 信息源规划 → 3. 批量抓取
                                              ↓
6. 总结报告 ← 5. 交叉验证 ← 4. 内容分析
```

## 四、实战检查清单

每次调查前，确认你覆盖了：

- [ ] 问题定义清晰，知道要找什么
- [ ] 规划了至少 3 类不同信息源
- [ ] 包含官方渠道 + 第三方独立来源
- [ ] 准备了 URL 列表，使用批量抓取
- [ ] 有交叉验证的计划
- [ ] 考虑了"历史快照"等存档手段
- [ ] 识别了可能的利益冲突方
- [ ] 记录了矛盾点并尝试解释
- [ ] 结论有证据支撑，不做过度推断
