# 调查方法论

## 一、问题定义

在开始调查前，先明确：

| 要素 | 说明 | 示例 |
|------|------|------|
| **核心问题** | 你到底想知道什么？ | "X 公司是否真的裁员 30%？" |
| **时间范围** | 关注什么时间段的信息？ | "最近 1 个月" |
| **地理范围** | 关注哪些地区？ | "中国区" |
| **信息类型** | 需要什么类型的证据？ | "官方公告、员工爆料、财经数据" |

## 二、信息源选择策略

| 信息类型 | 优先选择 | 避免 |
|---------|---------|------|
| **事实数据** | 官方公告、财报、权威媒体 | 匿名论坛、小道消息 |
| **市场反应** | 股价、交易量、预测市场 | 单一自媒体解读 |
| **民间声音** | 多个独立信源交叉验证 | 单个"内部人士" |
| **历史记录** | Wayback Machine、存档站 | "我记得"式口述 |

## 三、调查流程

```
1. 问题定义 → 2. 信息源规划 → 3. 批量抓取
                                              ↓
6. 总结报告 ← 5. 交叉验证 ← 4. 内容分析
```

### ⚠️ 强制要求：跨渠道交叉验证

**每项关键结论，必须有 2+ 不同类型来源交叉确认。**

| 结论来源类型 | 交叉验证价值 |
|------------|-------------|
| 权威媒体 + 预测市场 | ✅✅ 强：官方叙事 vs 用钱投票 |
| 权威媒体 + 社交平台 | ✅✅ 强：精英叙事 vs 民间观察 |
| 搜索引擎 + 垂直社区 | ✅ 中：大众信息 vs 专业深度 |
| 同类型平台（如两个论坛） | ⚠️ 弱：同质来源，难以交叉验证 |

**渠道差异越大，验证越可信。** 不要在同类平台上重复搜索就停止。

**自问清单：**
- [ ] 这个结论有 2+ 不同类型来源支持吗？
- [ ] 我用的渠道是否足够多样化（权威媒体/社交/市场数据/垂直社区）？
- [ ] 有没有刻意去"反向验证"——找和主流说法矛盾的来源？

## 四、实战检查清单

每次调查前，确认你覆盖了：

- [ ] 问题定义清晰，知道要找什么
- [ ] 规划了至少 3 类不同信息源
- [ ] 包含官方渠道 + 第三方独立来源
- [ ] 准备了 URL 列表，使用批量抓取
- [ ] 有交叉验证的计划
- [ ] 考虑了"历史快照"等存档手段
- [ ] 识别了可能的利益冲突方
- [ ] 记录了矛盾点并尝试解释
- [ ] 结论有证据支撑，不做过度推断

## 五、市场反应型任务专项清单

当调查目标是**事实核查型**（如"X事件是真的吗"、"Y走势会如何"），市场反应是核心指标，**优先级仅次于权威媒体报道**。

### 5.1 必须覆盖的市场维度

| 市场类型 | 为什么重要 | 关键信号 |
|---------|----------|---------|
| **预测市场**（ Polymarket / Manifold / Kalshi） | 用钱投票，比表态更真实 | 概率变化、交易量、争议性高的选项 |
| **股市/加密** | 立即反映机构真实预期 | 涨跌幅、异常交易量、相关板块联动 |
| **大宗商品**（原油/黄金） | 地缘危机的晴雨表 | 价格波动幅度、突破关键阻力位 |
| **债券/汇率** | 避险情绪指标 | 美债收益率、美元/瑞郎走势 |

### 5.2 分析框架

**第一步**：找到相关的预测市场问题（如 Manifold 上 "Will Iran war end by June 2026?"）
**第二步**：记录概率和交易量——高交易量 + 高概率 = 市场共识
**第三步**：特别关注**低概率但高交易量**的选项——这是"聪明钱"在押注

**信号强度判断：**
- ✅✅✅ 高交易量 + 高概率变化 + 与基本面一致
- ✅✅ 中交易量 + 单向概率变化
- ✅ 低交易量 + 概率波动大 = 噪音居多

### 5.3 常见误区

- ❌ 只看媒体报道，不看市场信号 → 错过机构真实看法
- ❌ 把低交易量市场的概率当真 → 预测市场需要流动性支撑
- ❌ 把市场预测当定论 → 市场会犯错，尤其在尾部风险上

## 六、产品全景图类调查专项清单

当调查目标是**找全所有同类产品/服务**（如"Coding Plan 有哪些平台"）时，必须执行以下步骤：

### 6.1 搜索引擎多轮搜索

**第一轮：找已知玩家**
```
搜索词：{产品名} 订阅 套餐
示例：Coding Plan 订阅 套餐
```

**第二轮：找"还有谁"**
```
搜索词：{产品名} 还有哪些平台
搜索词：{竞品A} {竞品B} {产品名} 对比
搜索词：国内 {产品名} 平台 有哪些
```

**第三轮：反向验证**
```
搜索词：{产品名} 评测 对比 哪个好
搜索词：用了 {产品名} 怎么样
```

### 6.2 权威来源覆盖检查

每个平台至少需要通过 **2 个独立来源** 确认其存在，不可只靠一个搜索结果。

| 平台类型 | 必须查证 |
|---------|---------|
| 云厂商（阿里/腾讯/字节/百度） | 官方产品页 + 第三方报道 |
| 创业公司（月之暗面/智谱） | 官网 + 科技媒体 |
| 运营商（联通等） | 官方公告 + 社区讨论 |

### 6.3 常见遗漏原因与应对

| 常见遗漏 | 应对方法 |
|---------|---------|
| 搜索词不够宽 | 用"还有哪些平台"类搜索补全 |
| 官网改版/404 | 尝试 `site:zhihu.com` 或科技媒体 |
| 页面需要登录 | 换第三方报道来源 |
| 反爬虫拦截 | 换 Bing 国际版，或用 `textise dot iitty` 参数 |
| 内容被删除 | 查 Wayback Machine 历史快照 |

### 6.4 交叉验证要求

在最终报告前，必须自问：
- [ ] **找到的平台数量是否达到预期？**（如：预期5-8家，实际找到几家？）
- [ ] **每个平台是否都有2+独立来源？**
- [ ] **有没有"这家也有这个产品"的线索被忽略？**（如：文章提到"Kimi 也推出了 Code Plan"）
- [ ] **有没有平台被遗漏是因为"名称不熟悉"？**（如：ArkClaw 就是字节跳动的火山引擎）

## 七、实时事件调查专项（正在发生的事）

当调查目标是**正在发生的事件**（如"Mojtaba 是否已死"、"某地是否发生政变"）时，标准调查流程完全不适用。必须换一套方法。

### 7.1 实时事件的信息层级

| 层级 | 证据类型 | 可信度 | 特点 |
|------|---------|--------|------|
| **L1** | 权威媒体正式报道 | ⭐⭐⭐⭐⭐ | 有编辑审核，最可靠 |
| **L2** | 预测市场概率 | ⭐⭐⭐⭐ | 用钱投票，但需要高流动性 |
| **L3** | 金融市场数据 | ⭐⭐⭐⭐ | 里亚尔/黄金/股市立即反映 |
| **L4** | YouTube/视频证据 | ⭐⭐⭐ | 需要鉴别真伪 |
| **L5** | 社交平台爆料 | ⭐⭐ | 极度不确定，必须交叉验证 |

**核心原则：L1 出现前，L2/L3 是最重要的信号。**

### 7.2 实时事件必查维度

**每项实时事件，必须查这4个维度：**

1. **预测市场**（最重要！）
   - Polymarket / Manifold 上有没有相关问题
   - 概率是多少？交易量够不够大？
   - 特别注意：低概率+高交易量 = 聪明钱在押注

2. **金融市场**（立即反映）
   - 货币汇率（如果事件涉及某个国家）
   - 黄金/原油（地缘危机晴雨表）
   - 股市相关板块
   - **看"今天"的变化，而不是绝对值**

3. **官方渠道**（ground truth）
   - 该国官方媒体网站（即使被封锁也要试）
   - 政府新闻发布会
   - 外交渠道声明

4. **视频/图像证据**（需要鉴别）
   - YouTube 上有没有现场视频
   - 发布时间是什么时候
   - 发布者的可信度

### 7.3 实时事件调查自问清单

- [ ] **预测市场有没有相关问题？** 概率和交易量是多少？
- [ ] **相关货币/黄金/股市今天有什么异常波动？**
- [ ] **该国官方媒体今天发了什么？**
- [ ] **有没有视频证据？发布者是谁？发布时间？**
- [ ] **各国政府/外交部的表态是什么？**
- [ ] **有没有"沉默"本身就是信号？**（如：一周不露面 = 重伤或死亡）

### 7.4 实时事件的置信度判断

| 情况 | 置信度 | 说明 |
|------|--------|------|
| 预测市场83% + 货币升值8% + 政权正常运转 | ⭐⭐⭐⭐⭐ | 高度确定 |
| 预测市场60% + 媒体矛盾报道 | ⭐⭐⭐ | 不确定，需要更多证据 |
| 只有社媒爆料 + 无市场信号 | ⭐⭐ | 极不确定 |

## 八、多维度证据链构建

对于重大、争议性强的事件，必须构建**多维度证据链**——不同类型的证据相互印证，才能提高置信度。

### 8.1 证据类型矩阵

| 证据类型 | 示例 | 在证据链中的角色 |
|---------|------|---------------|
| **官方叙事** | 总统声明、政府公告 | 代表官方说法 |
| **市场信号** | 预测市场、货币、黄金 | 用钱投票，最客观 |
| **精英观察** | 媒体分析、专家解读 | 提供背景和框架 |
| **民间信号** | 社交媒体爆料、民间视频 | 最接近现场，但最难核实 |
| **历史先例** | 类似事件的演变规律 | 提供判断参照 |

### 8.2 证据链的强度判断

**强证据链：L2 + L3 + L4 相互印证**
> 例子：预测市场显示83%活着 + 里亚尔升值8% + YouTube出现官方视频

**弱证据链：只有L5单来源**
> 例子：只有一个X.com帖子称"他已经死了"，没有任何其他证据支持

### 8.3 交叉验证的层次

不要满足于"多个媒体说了同一件事"——要刻意去找**不同类型的证据**：

```
✅ 强：权威媒体报道 + 预测市场信号 + 金融市场反应
✅✅ 强：以上三者 + YouTube现场视频 + 官方声明
⚠️ 弱：多个匿名社媒账号说了同样的事（同类来源，无法交叉验证）
❌ 无效：只靠"我听说"
```

### 8.4 识别"沉默的信号"

有些最重要的证据是**没有发生的事**：
- 某重要人物长期不露面 → 健康或权力出问题
- 某官方机构突然沉默 → 内部动荡
- 市场没有预期中的反应 → 说明内部消息已经被定价
- 对手国突然改变语调 → 可能有幕后交易
